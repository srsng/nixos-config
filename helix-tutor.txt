=================================================================
=                         欢迎使用 HELIX                        =
=================================================================

这是 Helix 编辑器的交互式中文教程。

本文件是官方 runtime/tutor 的本地汉化版。目标很简单：让你直接在
Helix 里练习移动、选择、编辑、多光标、寄存器、宏、匹配模式和窗口分割。

如果你已经用 hx --tutor 打开了这个文件，那么你就在正确的位置。
请按顺序完成每个小节。教程里的练习通常有一行以 --> 开头，你需要移动到
那一行，并按说明修改它。

重要约定：

 * Helix 是模态编辑器。普通模式用于移动和执行命令，插入模式用于输入文本。
 * Helix 的核心节奏通常是：先产生选择区，再对选择区执行操作。
 * Escape 通常用于回到普通模式。
 * Ctrl-x 表示同时按住 Ctrl 和 x。
 * Alt-x 表示同时按住 Alt 和 x。
 * Space 表示空格键。

不要急着一次读完。像练琴一样，把手指练熟比背命令更重要。

=================================================================

使用 h、j、k、l 移动光标。

        k
    h       l
        j

h 向左，j 向下，k 向上，l 向右。

1. 把光标移动到下面标记为 '-->' 的行。
2. 反复使用 h j k l，在这一行里移动。
3. 试着只用键盘移动，不要用鼠标。

--> 在这一行里练习 h j k l 移动光标。

提示：方向键也可以用，但 h j k l 是 Helix 用户最常用的移动方式。



=================================================================
=                         1.2 退出 HELIX                         =
=================================================================

输入 :q 退出 Helix。

冒号 : 会打开命令行提示符。输入 q 后按 Enter 即可退出。

1. 输入 :q 然后按 Enter。
2. 如果你不想真的退出，可以按 Escape 取消命令行。

如果文件有未保存的修改，Helix 会阻止你直接退出。这时可以使用 :q! 强制退出并丢弃修改。

2. 把光标移到多余字符上。
3. 输入 d 删除它。
4. 重复直到这一行和下一行一致。

--> Heliix 是一个很棒棒的编辑器。
    Helix 是一个很棒的编辑器。

提示：如果误删了，后面会学撤销。

=================================================================

输入 i 进入插入模式。

在插入模式中，你输入的普通字符会被写入文件。按 Escape 回到普通模式。

 * :q!  强制退出，不保存

1. 输入 :w 并按 Enter。
2. 看状态栏提示，确认文件已经保存。

=                    3.1 移动命令与选择区                        =
=================================================================

输入 w 向前选择到下一个单词开头。

d 键并不是“删除光标处的字符”这么简单；它删除当前所有选择区。你的光标可以看作一个单字符选择区。

1. 移动到下面标记为 '-->' 的行。
2. 移动到一个需要删除的多余单词开头。
3. 输入 w 选中到下一个单词开头。
4. 输入 d 删除选择区。
5. 重复，删掉所有多余单词。

--> 这个 句子 铅笔 有 真空 多余 的 词语。
    这个句子有多余的词语。

提示：Helix 常见节奏是“移动产生选择，然后执行操作”。



=================================================================
=                       3.2 更多移动命令                         =
=================================================================

输入 w 选择到下一个单词开头。
输入 e 选择到当前或下一个单词结尾。
输入 b 选择到前一个单词开头。

大写 W、E、B 的含义类似，但按 WORD 移动。WORD 以空白分隔；word 更细，会把标点也纳入边界判断。

1. 移动到下面标记为 '-->' 的行。
2. 使用 w、e、b 在行内来回移动。
3. 观察每次移动后被选中的范围。
4. 使用 W、E、B 再试一次。

--> Helix 是一个 one-of-a-kind "modal" text editor

提示：小写移动更细，大写移动更粗。



=================================================================
=                     3.3 WORDS 和 words                         =
=================================================================

word 通常指由字母、数字或符号切分出来的较小单位。WORD 指由空白分隔出来的较大单位。

例如：one-of-a-kind 可能需要多次 w 才能走完，但通常只需要一次 W 就能作为一个整体跨过去。

1. 移动到下面标记为 '-->' 的行首。
2. 反复输入 w，观察选择如何逐步跨过单词和标点。
3. 回到行首。
4. 反复输入 W，观察 WORD 移动的差异。

--> Helix is a one-of-a-kind "modal" text editor

提示：写代码时，小写移动适合精确编辑，大写移动适合快速跨段。

=                       3.4 修改命令                             =
=================================================================

输入 c 修改当前选择区。

c 会删除当前选择区并进入插入模式，所以它等价于“删除后立刻插入”。这是非常常用的快捷方式。

1. 移动到下面标记为 '-->' 的行。
2. 移动到错误单词开头，输入 e 选中它。
3. 输入 c 删除该词并进入插入模式。
4. 输入正确单词。
5. 按 Escape 回到普通模式，重复直到两行一致。

--> 这张 纸 有 沉重 的 词 在 后面。
    这个句子有错误的词在里面。

提示：如果要改一个词，常见手法是移动到词首，按 e，再按 c。

3. 输入 3e 移动到后面第 3 个 word 的结尾。

--> 这是一行可以让你反复移动练习的文字。

3. 用 w 选中 oranges。
4. 输入 R，用 watermelons 替换 oranges。

--> I like watermelons because oranges are refreshing.
    I like watermelons because watermelons are refreshing.

提示：R 是 replace with yanked text，不会进入插入模式。

=                         8.1 寄存器                             =
=================================================================

寄存器是用单个字符命名的存储容器，可以保存复制、删除、宏等内容。

输入 "<寄存器> 选择接下来命令使用的寄存器。例如 "ay 会把当前选择区复制到 a 寄存器。

1. 移动到下面标记为 '-->' 的行。
2. 输入 w 选中 alpha。
3. 输入 "ay，把 alpha 复制到 a 寄存器。
4. 移动到 beta 上，输入 "ap，把 a 寄存器内容粘贴出来。
5. 尝试用另一个寄存器保存不同文本。

--> alpha beta gamma

常用寄存器：

 * "  默认寄存器。
 * _  黑洞寄存器，写入它的内容会被丢弃。
 * +  系统剪贴板寄存器。
 * /  搜索寄存器。

提示：寄存器让“复制 A、再复制 B、稍后仍能粘贴 A”成为可能。

5. 移到下一行，输入 q 重放宏。

--> word one
--> word two
--> word three

提示：宏适合重复结构非常相似、但又不值得写脚本的编辑。

3. 回到本节，试试 gh、gl、gs。

=                       13.1 窗口分割简介                        =
=================================================================

Helix 可以把编辑区分成多个 split，同时查看多个文件或同一文件的不同位置。

分割窗口在阅读代码、对照修改、查看定义和实现时非常有用。

1. 在本节阅读 split 命令。
2. 如果你正在终端里练习，可以实际打开一个新 split。
3. 不想改变当前界面时，只阅读命令即可。

提示：split 是工作区布局，不是复制文件。



=================================================================
=                       13.2 垂直和水平分割                      =
=================================================================

输入 Ctrl-w v 创建垂直分割。
输入 Ctrl-w s 创建水平分割。

1. 输入 Ctrl-w v，创建左右分割。
2. 输入 Ctrl-w s，创建上下分割。
3. 用 :q 关闭当前 split。

提示：Ctrl-w 是窗口命令前缀，很多 split 操作都从它开始。



=================================================================
=                       13.3 在 split 间移动                     =
=================================================================

输入 Ctrl-w h / j / k / l 在分割窗口之间移动。

 * Ctrl-w h 移到左侧 split。
 * Ctrl-w j 移到下方 split。
 * Ctrl-w k 移到上方 split。
 * Ctrl-w l 移到右侧 split。

1. 创建两个或多个 split。
2. 用 Ctrl-w h/j/k/l 在它们之间移动。
3. 用 :q 关闭多余的 split。

提示：这和基本移动键 h/j/k/l 的方向一致。



=================================================================
=                    13.4 使用命令创建 split                     =
=================================================================

也可以用命令打开分割窗口：

 * :vsplit 或 :vs  垂直分割当前文件。
 * :hsplit 或 :hs  水平分割当前文件。
 * :vsplit <文件名>  在新垂直 split 中打开文件。
 * :hsplit <文件名>  在新水平 split 中打开文件。

1. 输入 :vs 并按 Enter。
2. 输入 :q 关闭新 split。
3. 如果愿意，试试 :hs。

提示：命令形式适合和文件路径一起使用。



=================================================================
=                       13.5 交换 split                          =
=================================================================

Helix 可以交换当前 split 与相邻 split 的位置。

常见窗口交换命令在 Ctrl-w 模式下完成，具体键位可以用 Space ? 搜索 split 或 window 查看。

1. 创建两个 split。
2. 打开命令面板，搜索 split / window。
3. 找到交换 split 的命令并尝试执行。

提示：不同版本的键位可能会变化；命令面板能显示当前 Helix 实际支持的命令。



=================================================================
=                       13.6 转置 split                          =
=================================================================

转置 split 会改变 split 的布局方向，例如把左右关系转换成上下关系。

1. 创建几个 split。
2. 使用命令面板搜索 transpose。
3. 执行相关命令，观察布局变化。

提示：当布局越来越乱时，关闭多余 split 再重新分割通常更快。



=================================================================
=                  13.7 从文件选择器打开 split                   =
=================================================================

在文件选择器里，可以直接把选中文件打开到新的 split：

 * Ctrl-v 在新的垂直 split 中打开选中文件。
 * Ctrl-s 在新的水平 split 中打开选中文件。

1. 按 Space f 打开文件选择器。
2. 选中一个文件。
3. 按 Ctrl-v 或 Ctrl-s 打开到新 split。
4. 用 :q 关闭不需要的 split。

提示：这比先打开文件再手动分割更顺手。



=================================================================
=                        第 13 章回顾                            =
=================================================================

 * Ctrl-w v 创建垂直 split。
 * Ctrl-w s 创建水平 split。
 * Ctrl-w h/j/k/l 在 split 间移动。
 * :vs 和 :hs 可以通过命令创建 split。
 * 文件选择器中 Ctrl-v / Ctrl-s 可以直接打开到新 split。
